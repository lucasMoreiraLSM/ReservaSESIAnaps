Por que não consigo excluir / alterar uma reserva?

Para excluir ou alterar uma reunião, você deve estar logado com o mesmo nome de usuario e senha que foi utilizado pela pessoa que marcou a reunião. Outra solução é entrar em contato com um dos administradores ou a pessoa que inicialmente fez a reserva para excluir ou alterar.

Como crio uma reserva?

Clicar no horário desejado leva você para a tela de reserva. Preencha os detalhes e clique em "Salvar".

Em vez de clicar no horário desejado, você pode selecionar um intervalo de tempo e também uma sala ou intervalo de dias, arrastando o cursor sobre um grupo de células no visualizações de dia ou semana. Quando o mouse for liberado, você será levado à tela de reserva, horários e a sala ou o período do dia preenchido por você.

Qual é a diferença entre quinto e último?

Ao fazer uma reserva mensal recorrente, o quinto representa exatamento o quinto dia da semana de cada mês. Se um mês não tiver um quinto dia da semana, nenhuma reserva será feita para esse mês. Se você escolher o último dia da semana do mês, uma reserva será feita no quarto dia da semana, se não houver um quinto dia da semana.

Quinto últimofunciona de maneira semelhante, mas contando ao contrário a partir do final do mês. Usar primeirofornecerá o quarto último dia da semana do mê se não houver quinto último dia da semana.

Como agendar salas em locais diferentes?

Atualmente o sistema não pode reservar duas salas diferentes simultaneamente. Você deve agendar cada um separadamente. Certifique-se de que a hora desejada está disponível nos dois sites antes de fazer uma reserva.

Qual é a diferença entre InternaeExterna?

Por padrão, o Reserva SESI define dois tipos de reunião. Interno significa que o a reunião será assistida apenas por funcionários. Uma reunião externa também pode ter como participantes clientes, fornecedores, investidores, etc. Seu site pode definir até um total de 10 tipos de reservas, de acordo com suas necessidades. As reuniões são destacadas na tela principal do calendário com uma cor correspondente ao tipo e uma chave de cor de todos os tipos definidos é mostrada na parte inferior da tela principal do calendário.

Como o sistema funciona e quem o escreveu?

O Reserva SESI foi desenvolvido de forma voluntaria pelo estagiário da escola, Lucas Silva Acadêmico do curso de Sistemas de Informação UEG. E implementado no período de "Setembro/2019".
Visa a solução de problemas recorrentes como reservas conflitantes, falha na comunicação, excesso de documentação e papeis, afim de otimizar e facilitar todo o processo de reservas de ambientes na instituição.
Para o desenvolvimento, foi utilizado a linguagem de programação PHP,CSS,HTML e JavaScript, juntamente com scripts desenvolvidos por Meeting Room Booking System e Webcalendary.
Que é um script de código aberto distribuído sob a Gnu Public License (GPL). Isso significa que o software é livre para usar, distribuir e modificar. Desta mesma forma a Lsm.System disponibiliza o sistema Reserva SESI pela mesma licença! Mais informações contate o desenvolvedor!

O sistema está escrito principalmente em PHP, que é uma linguagem de programação de código aberto que pode ser incorporada na páginas Web semelhantes em conceito às páginas do servidor ativo da Microsoft. PHP é especialmente bom em acessar bancos de dados.

O banco de dados usado para o sistema é MySQL or PostgreSQL. MySQL é muito rápido, multiencadeado, multiusuário e robusto Servidor de banco de dados SQL (Structured Query Language) que também é GPL. O PostgreSQL é um SQL Relacional para Objetos de código aberto, multiusuário, com todos os recursos servidor de banco de dados.

O sistema será executado em várias plataformas, incluindo a arquitetura do PC usando o sistema operacional Linux . Linux, é um sistema operacional gratuito, de código aberto, semelhante ao unix.

O servidor da web que está sendo usado é mais uma peça de código aberto gratuito Programas. O servidor da web Apache é o servidor da web mais popular do mundo.
